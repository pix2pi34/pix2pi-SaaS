Pix2pi FAZ 4D-10

Paraşüt Discovery

Bu yüzey Pix2pi ERP core ile Paraşüt arasında kurulabilecek entegrasyon sınırlarını discovery seviyesinde göstermek için hazırlanmıştır. Gerçek API çağrısı, credential saklama, canlı fatura, canlı tahsilat ve production sync bu adımda yapılmaz.

Paraşüt Durumu: DISCOVERY Gerçek API Çağrısı: KAPALI Credential: YAZILMAZ Pix2pi ERP Core: ANA KAYNAK
01

Cari / Müşteri Eşleme

Pix2pi customer/party kayıtlarının ileride Paraşüt müşteri kayıtlarıyla nasıl eşleşeceği discovery seviyesinde tutulur.

02

Ürün / Hizmet Eşleme

Product/item kayıtlarının dış sistem ürün/hizmet karşılıkları ayrı mapping alanı olarak değerlendirilir.

03

Satış / Fatura Akışı

Satıştan fatura veya ön muhasebe çıktısına giden sınır keşfedilir; canlı fatura oluşturulmaz.

04

Tahsilat / Ödeme

Tahsilat ve ödeme hareketlerinin sync yönü sonraki fazda belirlenir; production ödeme işlenmez.

05

TDHP / UFK Koruması

Pix2pi TDHP ve UFK kararları korunur. Dış entegrasyon ERP core muhasebe kararlarını kontrolsüz ezmez.

06

e-Fatura / e-Arşiv

e-Belge production kapsamı ayrı tutulur. Bu adım sadece entegrasyon sınırı ve keşif kararıdır.

07

Tenant-aware Integration

Her tenant kendi bağlantı durumuna, mapping kayıtlarına ve sync geçmişine sahip olmalıdır.

08

Secret / Credential Policy

API key, token veya client secret repo içine yazılmaz. Secret yönetimi production güvenlik fazında ele alınır.

09

Marketplace Ayrımı

Pazar yeri satış kanalı ile ön muhasebe bağlantısı karıştırılmaz. Marketplace 4D-9 altında ayrı mühürlenmiştir.
